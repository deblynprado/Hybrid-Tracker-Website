# Hybrid Tracker Website - Project Rules

To ensure consistency and quality for the Hybrid Tracker project, the following rules apply to all agent interactions:

## 1. Technical Constraints
- **Stack**: Pure HTML5, CSS3, and Vanilla JavaScript.
- **Dependencies**: Zero external dependencies unless explicitly requested. No frameworks (React, Vue, Tailwind, etc.).
- **Structure**: Maintain a clean, flat directory structure.
- **Performance**: Optimize images and ensure fast load times.

## 2. Aesthetic Standards (Premium)
- **Design Style**: "Sophisticated Clarity". Use 2026 trends like fluid glassmorphism, kinetic typography, and intentional motion.
- **Micro-interactions**: Subtle hover states, smooth transitions, and gesture-friendly elements.
- **Typography**: Use high-quality Google Fonts (e.g., *Inter*, *Outfit*, or *SF Pro* alternatives).
- **Assets**: Use the `generate_image` tool for placeholders or specific visual needs to maintain a high-end look.

## 3. Deployment & Git Workflow
- **Commits**: Follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/). (e.g., `feat: add hero section`, `fix: privacy policy layout`).
- **Safety**: Never push experimental code. Verify all changes locally using the browser tool before committing.
- **Manual Sync**: Remind the user about manual deployment after significant milestones.

## 4. Content Specifics
- **App Store**: Strictly follow Apple's Marketing Guidelines (badges, spacing, branding).
- **Privacy**: The privacy policy must be easily accessible and formatted for readability.

## 5. Agent Verification
- At the start of each session, read this file to refresh context.
- Before finishing a task, cross-check against these rules.
